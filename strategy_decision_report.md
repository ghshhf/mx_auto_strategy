### 💰 现金（16万）→ **网格弹药池（v6.3 新增）**

> **用户核心要求（v6.3 变更）**："那种波动大的，咱们不是留了点现金吗？可以做网格呀，**每天要交易一次**的。不可能5天时间这么慢进去不动吧？"

之前现金是"躺着的"，现在改为**用16%现金做隔日网格**，对高波动进攻票（凯莱英）低买高卖吃差价。

| 网格参数 | 数值 | 说明 |
|---------|------|------|
| 弹药总量 | 16万 | = cash_reserve_pct 16% 全额启用 |
| 网格层数 | 5 层 | 每层 ≈ 3.2万 |
| 凯莱英步距 | **5.26%** | 动态算出（σ6.89%×0.8），波动大→格距宽 |
| 长江电力步距 | **2.00%** | 已达下限（波动小→格距窄） |
| 迈瑞/南网步距 | 2.00% | 备选网格标的 |
| 触发频率 | **每天1次** | 用户要求"每日交易"，不躺平 |
| T+1 假设 | 隔日反向 | 今天买的网格仓明天才能卖 → 天然隔日网格 |

**网格不碰防御底仓**（防御仓建一次不动）。网格只在进攻票（凯莱英/长江电力）上滚动，用现金池做低买高卖，震荡市持续产生小额收益，把"5天不动"变成"每天有交易"。

> **同收益率优先规则利好网格**：大赛规则"同收益率→交易笔数多者优先"。网格每天产生交易，正好在相同收益下比躺平的账户排名更靠前。

## 七、操作时间线（v6.3 更新：每日3次手动触发）

> **用户作息（明确）**：每天 **10:00 起床 / 12:00 看盘 / 14:00 看盘**，手动触发指令，**不装自动脚本、不常驻**。
> T+1 机制下，每日3次触发已覆盖开盘/盘中/尾盘，足够冲周名次，且无需盯盘。

| 你的时间点 | 触发指令 | 动作 |
|-----------|---------|------|
| **周一 10:00** | `python3.11 auto_trader.py --mode once` | 全新建仓（3防+2攻）+ 首轮网格 + 止损止盈检查 |
| **周二~周五 10:00** | `python3.11 auto_trader.py --mode grid` | 开盘后网格首轮（跌买涨卖）+ 持仓检查 |
| **每日 12:00** | `--mode grid` | 盘中网格二轮（T+1：早盘买的若触卖格可挂单） |
| **每日 14:00** | `--mode grid` | 尾盘网格三轮 + 当日收盘前最后再平衡 |
| **周五 14:00 后** | `python3.11 auto_trader.py --mode reset` | **周度结算清仓**（大赛规则：每周结算），清空底仓+网格状态 |
| **下周一 10:00** | `--mode once` | 新一周重新选股建仓（全新建仓循环） |

> **为什么每天3次交易**：① 你明确要求"每天交易、一周冲名次"；② 大赛"同收益率交易笔数多者优先"，每天3次网格让账户成交密集，相同收益下排名更靠前；③ 凯莱英σ6.89%波动大，多次网格把震荡转为累计收益。
> **T+1 是最优解而非妥协**：T+0 需全天常驻盯盘（你不愿），T+1 隔夜反向 + 每日定时3次触发，完美匹配你"10点起、懒得开电脑白天"的作息。

### 周度循环（核心节奏）

```
周一建仓 → 周二~周五每天3次网格 → 周五清仓 → 下周一重建（重复）
   ↑___________________________________________________↓
```

每周是一次独立冲刺：5个交易日定本周名次，周五清仓归零，下周重新选股。**不跨周持仓**，规避周末消息面跳空风险。
