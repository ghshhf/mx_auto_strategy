# mx_auto_strategy 回测报告（v6.10：16 倍框架 + 多指数周线死叉全防御）

## 摘要

本报告为 **v6.10 版本** 主线回测（2026-07-25 更新）。在此前真实持仓标的基础上，新增 **多指数周线死叉去风险层**：当 6 大宽基指数（沪深300/中证500/上证/创业板/上证50/中证1000）周线 MA5/MA20 同时熊化的指数 ≥3 时，触发「进攻仓清零、转全防御蓝筹篮」。

核心结论：**10 年起步 1 万元** 的 16 倍框架（防御蓝筹底仓 + 弱市进攻切可转债 + 周频再平衡）期末约 **16.1 万元**（+1509% / 年化 32.0% / 最大回撤 -23.7%）；叠加死叉全防御层后期末约 **16.8 万元**（+1577% / 年化 32.6% / 最大回撤 -20.9%）。即：**收益略升、回撤更浅**。买入持有沪深300 同期仅约 1.46 万元（+46% / 回撤 -45.6%）。

> 注意：本回测仍是**配置层代理**（进攻篮用真实代表标的，非逐笔重放），未计交易成本/滑点。详见「重要假设与局限」。

## 16 倍框架 = 什么？

- **防御端**：DEF16 红利赢家篮（长江电力、招商银行、中国神华、宁沪高速、贵州茅台、伊利股份等）。
- **进攻端**：OFF4 核心精选（宁德时代、中际旭创、北方华创、凯莱英）。
- **市况切换**：`REGIME_ALLOC`（弱/平/强）按周频再平衡；**弱市时进攻仓自动切换为可转债**（南银转债 113050、兴业转债 113052，债底保护）。
- **止盈/止损/阶梯止盈**：沿用仓库 `strategy_config.json` 原参数（硬止损 -5% / -10%、阶梯止盈等）。

这一框架本身在 10 年窗口跑出约 16 倍，是后续叠加的基础。

## 死叉全防御层 = 什么？

新增 `death_cross.py` 模块：

- 监测 6 大宽基指数周线：沪深300（sh000300）、中证500（sh000905）、上证指数（sh000001）、创业板指（sz399006）、上证50（sh000016）、中证1000（sh000852）。
- 单指数「熊化」定义：收盘价 < 20 周均线，且 5 周均线 < 20 周均线，且 20 周均线向下。
- 当 **≥3 个指数同时熊化**（`threshold=3`）时触发去风险信号。
- `auto_trader.run_once` 在信号触发时：
  - 当周 **不新增进攻仓位**；
  - 释放的进攻额度转入防御蓝筹篮；
  - **已持仓的进攻票仍按原有止盈/止损持有**（不强行一键清仓，遵循「周频不全卖」的铁律）。
- 任一指数抓取失败且可用比例 < 80% 时，**保守不触发**（安全降级）。

## 死叉层为什么是「去风险层」而不是收益引擎？

从表可见：

- **10 年**：新主线比无死叉版多赚约 +4.2% 相对收益，回撤少 2.8pp（-23.7% → -20.9%）。这是因为 2018 / 2022 等系统性下跌段，提前把进攻仓切进防御篮，避开了部分高 Beta 崩溃。
- **5 年**：新主线收益略低于无死叉版（31,730 vs 33,521）。原因是 2024-2026 的反弹/震荡市中，死叉信号偶尔滞后，导致少参与一段进攻反弹。这是「以收益换平滑」的典型表现。
- **3 年**：新主线同样略低（25,248 vs 27,950），道理同上。

因此，死叉层的定位是**风险调速器 / 防御端平滑器**：长期可能微提收益+压回撤，但短期牛市/震荡市会小幅牺牲收益。不应把它当 alpha 引擎去调参。

## 重要假设与局限（看数前必读）

- **仍是配置层代理，非逐笔交易重放**。真实 alpha 来自 `weekly_theme.pick_theme()` 每周动态选的 2 只进攻股，该链路无历史数据源可复现；平衡/强势进攻篮用的是 OFF4 真实高 Beta 代表样本（宁德/中际旭创/北方华创/凯莱英），其波动/收益不等于系统实际每周选股结果。
- **未计交易成本/印花税/滑点**：周频再平衡的年化摩擦约 0.5%~1.5%，会进一步侵蚀上表收益。
- **单一样本区间依赖**：10 年区间包含 2018/2022 多轮熊市 + 2019-2021 结构牛 + 2024-2026 红利/科技修复，结论仍受样本区间依赖；严格结论需 walk-forward / 样本外验证，切忌过拟合。
- **信号滞后风险**：死叉是趋势确认型信号，在急跌中段触发，可能在 V 型反转的反弹初期错过；5 年/3 年样本中已体现这种「削顶」成本。
- **两融维度不可得**：温度探针的「融资余额」维度在标的历史中无法获取，按方案 C 稳健设计「维度失败即排除并权重重归一化」处理，不影响安全结论。

## 对实盘的含意

- 框架本身（防御蓝筹为底 + 弱市进攻切转债 + 周频再平衡）在 2016-2026 这个覆盖多轮牛熊的 10 年窗口显著跑赢宽基，且回撤大幅更低——支撑「守正出奇」的主线设计。
- 死叉全防御层作为**风险调速器**，适合在系统性下跌段替你收力；建议默认启用，但不要期待它显著增收。
- 若想让 5 年/3 年短期收益更贴近无死叉版，可把 `death_cross.threshold` 从 3 提高到 4 或 5，但会牺牲部分防御效果；反之降到 2 会防御更积极、也可能更频繁削反弹。
