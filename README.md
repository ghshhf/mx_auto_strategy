# mx-moni 自动交易策略 (v4)

> 基于东方财富妙想模拟组合管理 skill (mx-moni) 的条件触发自动买卖框架
> 模拟盘专用：第17期龙虾炒股大赛（2026.07.20 - 07.24，一周结算）
> **v4 基于下跌市回测升级**：趋势过滤 + 科技软约束 + 防御模式降仓位 + 跌停友好止损

## 名次目标与策略映射（你给的标尺）

| 收益目标 | 比赛名次 | 奖金 | 策略姿态 |
|---------|---------|------|---------|
| 不亏 / 微赚 | > 1000 名 | 0.8 元 | **保底线**：绝不能掉出，靠分散+不满仓防回撤 |
| **+10 个点** | **≈ 前 10 名** | **50 元** | **主目标**：阶梯止盈前期少卖，留 80% 仓位吃满 +10% 区间 |
| 冲顶 | 第 1 名 | 500 元 | 弹性上限：留 12.5% 尾仓，万一某只成妖股分一杯 |

> 本质：**先保名次（>1000名=0.8元），再冲前10（+10%≈50元）**。不凹第一，但保留吃极端行情的尾仓。


## 策略核心（v4，基于回测升级）

| 维度 | 规则 |
|------|------|
| 选股 | 自动从候选池选股，**低PE + 高热度 + 历史低位** 三维评分 |
| 行业约束 | 5 只标的**强制跨 ≥4 个不同行业**；科技为**软偏好**（下跌市可 0 只，回测证伪"必须科技"）|
| 趋势过滤 | 沪深300 20日MA 向下 → **防御模式**（自动切换，无需人工判断）|
| 仓位 | 进攻：底仓 **70%** + 机动 30%；防御：底仓 **50%** + 机动 50%（不加仓，留现金等恐慌低点）|
| 加仓 | 仅进攻模式：回撤 ≥3% 或突破 +3% 加仓（单只最多 2 次）；防御模式不加仓 |
| 卖出 | **阶梯止盈（保名次 + 冲前10）**：+8%卖20% / +15%卖45% / +25%卖75% / +40%卖88% |
| 止损 | **-5% 硬止损**（防御）/ -8% 硬上限（进攻）；**跌停封死时锁定不补刀**，等次日开盘清 |
| 周期 | 一周结算一次（5 交易日），周末做评估 |

> 📊 **回测详情见 [backtest_report.md](./backtest_report.md)**：最近两周下跌市实证，科技股平均 -4.78%/回撤11.2% vs 非科技 +0.77%/回撤4.4%。

## 使用方法

```bash
cd /workspace/mx_auto_strategy

# 1) 先看自动选股结果（不交易，安全）
python3.11 auto_trader.py --mode select

# 2) 单次执行（选股 + 条件买入 + 止盈，会真下单！）
python3.11 auto_trader.py --mode once

# 3) 盘中自动循环（交易时段每5分钟检查一次，全自动）
python3.11 auto_trader.py --mode loop

# 4) 仅检查持仓止盈
python3.11 auto_trader.py --mode sell
```

> ⚠️ 开赛前请用 `--mode select` 反复验证选股；确认策略无误后，开赛日 09:30 起用 `--mode loop` 全自动运行。
> ⚠️ `--mode once` / `--mode loop` 会**真实调用 mx-moni 下单**，模拟盘资金会变动，请知悉。
> 🔒 已加入**交易时段硬守卫**：非交易时段（周末/午休/收盘后）任何真实下单指令都会被自动拦截，不会发出无效委托。

## 关于「没开盘能不能排单」

**结论：这个模拟盘不支持闭市预埋单。**

实测（周六 12:22 非交易时段）：
- ✅ **查询类接口可用**：`我的资金` / `我的持仓` 正常返回（确认账户 261984600000041416，总资金 100 万）。
- ❌ **下单接口闭市被拒**：发送 `买入 000725 100 股` 返回 `status:100 输入价格不合理,超过限价`。
  - 原因：下单必须带「合理价格」，而合理价格依赖**实时行情**；闭市时腾讯实时接口返回空，脚本拿不到当前价 → 触发服务端限价校验失败。
  - 与真实 A 股的「夜市委托/盘前预埋单」不同——这里闭市排不进，必须**开盘时段（工作日 9:30-11:30 / 13:00-15:00）**才能成交。
- ⏰ **盘前待验证**：9:00-9:25 集合竞价时段腾讯是否有实时价，待周一开盘前补测。

## 比赛目标对齐（赚个几毛钱）

> 一周结算一次 → 每天可交易；**排名 > 1000 名即可获 0.8 元奖励** → 目标=保住前 1000 名 + 小赚，不做激进操作。

| 维度 | 配置 | 为何服务目标 |
|------|------|-------------|
| 选股 | 行业冠军制（每行业只 1 只）+ 科技保底 | 分散风险，防单行业崩导致掉榜 |
| 仓位 | 70% 底仓 + 30% 机动 | 不满仓，留机动应对回撤 |
| 卖出 | 每涨 5% 卖 1/3（越长越卖） | 落袋为安，锁定小利 |
| 加仓 | 回撤≥3% / 突破+3% 才加 | 不追高，控制回撤 |
| 周期 | 每日交易、每周结算 | 匹配比赛节奏 |

## 环境变量（已永久配置）

- `MX_APIKEY`：已在 `~/.zshrc` 和 `~/.bashrc` 写入，永久生效
- `MX_API_URL`：默认 `https://mkapi2.dfcfs.com/finskillshub`

## 可调参数（strategy_config.json）

- `risk.base_position_pct`：底仓占比（默认 70）
- `risk.flex_position_pct`：机动资金占比（默认 30）
- `auto_select.top_n`：选几只（默认 5）
- `auto_select.min_industries`：最少行业数（默认 4）
- `auto_select.require_tech`：是否强制含科技（默认 true）
- `scoring.buy_score_min`：选股评分阈值（默认 0.55）
- `sell_rules.tiers`：阶梯止盈表，每档 `gain_pct`(触发涨幅) + `sell_portion`(累计卖出占总仓绝对比例)。默认 +8%/20%、+15%/45%、+25%/75%、+40%/88%
- `auto_select.candidate_pool`：候选股票池（含 industry 和 tech 标记）

## 已知限制

1. 行情来自腾讯财经**免费接口**，非交易时段（周末/休市）实时价/PE 取不到，仅 K 线分位可用。
2. `mx_moni.py` 自带的资金展示有 `/1000` 显示 bug（实际 API 返回值正确，不影响交易）。
3. 候选池为静态龙头子集（39 只），非全市场扫描。如需扩池，往 `candidate_pool` 加代码即可。
4. **闭市不能下单/排单**（见上文），所有真实交易必须落在交易时段内。
5. 成本基准 `_cost_basis` 现在已持久化到 `.cost_basis.json`（跨重启保留），首次运行前该文件不存在时视为空仓。
